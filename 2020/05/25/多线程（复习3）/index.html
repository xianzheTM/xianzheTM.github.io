多线程。

创建多线程的方式

继承Thread类

1.创建于继承于Thread类的子类

2.重写子类的run()方法–>将此线程要做的事写在run方法中

3.创建子类对象

4.调用子类的start()方法

package com.ylxianzhe.thread;

public class Test {
    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        myThread.start();//启动线程，调用run()方法
        //再启动一个线程不能再调一遍start方法，它内部有个threadstatus，start调用过后就改变了，没法再用
        //直接新建一个MyThread对象即可。
        for (int i = 0; i < 100; i++) {
            if (i % 2 != 0) {
                System.out.println(i+"********main**********");
            }
        }
    }
}

class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 0; i < 100; i++) {
            if (i % 2 == 0) {
                System.out.println(i);
            }
        }
    }
}

两个线程同时运行，具体如何执行要看CPU的调度。

package com.ylxianzhe.thread;

用匿名内部类，实现一个thread的匿名子类并且重写run方法。

package com.ylxianzhe.thread;

public class Test{
    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        myThread.start();
    }
}

class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 0; i < 100; i++) {
            if (i % 2 == 0) {
                System.out.println(Thread.currentThread().getName() + ':' + i);
            }
            if (i % 20 == 0) {//线程让步的意思，释放当前CPU的执行权，但并不带表让给另一个线程。下一刻可能又被抢回来了
            //Java14必须要写明yield()的调用者，以前可以不写
                MyThread.yield();
            }
        }
    }
}
public class Test {
    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        myThread.start();
        for (int i= 0; i < 100 ; i++){
            if (i % 2 != 0) {
                System.out.println(Thread.currentThread().getName() + ':' + i);
            }
            if (i == 20) {
                try {
                    myThread.join();//在main线程里调用myThread.join()，意思是让main进入阻塞状态，先让myThread线程执行
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

还有个方法stop()。终止当前线程，已过时。

还有sleep()方法，sleep(1000)代表线程停一秒。一秒钟之内，CPU无法执行该线程，一秒后可以执行，但不一定是马上执行。使用时要用try-catch处理异常。

还有个isAlive()方法，判断线程是否存活。

线程的调度

多线程采用时间片的额策略，来回在几个线程中切换。这就涉及到优先级问题。

MAX_PRIORITY:10,

MIN_PRIORITY:1,

NORM_PRIORITY:5.

可以使用getPriority()返回线程优先级，setPriority()设置优先级。线程创建时继承父线程的优先级。
低优先级只是得到调度的概率低，不是绝对在高优先级后执行。

实现Runnable接口

1.创建一个实现了Runnable接口的类

2.实现类中要实现Runnable接口的抽象方法run()

3.创建实现类对象

4.将该对象传入Thread类的构造器，并创建Thread类对象

5.通过Thread类对象调用start()

public class Test {
    public static void main(String[] args) {

        new Thread(new MyThread()).start();
        new Thread(new MyThread()).start();
    }
}

class MyThread implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i < 100; i++) {
            if (i % 2 == 0) {
                System.out.println(Thread.currentThread().getName() + ':' + i);
            }
        }
    }
}

一般开发中用实现接口的方式，可以避免单继承的缺点，同时由于一个Runnable接口实现类传给多个Three构造器，实现类中的数据天然就是共享的。共同点是两种方法都要重写run()方法。

线程的生命周期

Thread.State定义了线程的多个状态。State是一个枚举类，里面有五个对象。

五种状态：

新建（没有开始start()方法），

就绪（开始了start()方法，可以执行，但还没分配CPU资源），

运行（分配到CPU资源，正在执行），

阻塞（被挂起，或者用了join()方法，但不是切换到另一线程执行的意思，那种还是就绪，而这种是绝对的不能执行，即使CPU资源分配了），

死亡（执行完run()，调用stop()，出现异常且没处理）。
当阻塞结束后是回到就绪状态，而不是运行状态。
触发阻塞的方法：sleep()，join()，等待同步锁，wait(),suspend(),(suspend过时)。阻塞不是线程的最终状态，它必定会回到就绪，最终是死亡。

回到就绪状态：sleep时间到，join结束，获取到同步锁，notify()/notifyAll()唤醒，resume()（已过时）

线程同步

多个线程同时处理一个数据就会出现线程安全问题。必要一个线程a处理完数据后才能让另一个线程b处理。即使线程a出现了阻塞也不能切换给线程b。

方式一：同步代码块

用synchronize关键字。

synchronized(同步监视器){//同步监视器，俗称锁，任何一个类的对象都可充当锁
    //要被同步的代码，即操作共享数据的代码
}

这个锁必须只能有一把，即每个线程都是这把锁

class MyThread implements Runnable {
    private int ticket = 100;
    Object lock = new Object();

    @Override
    public void run() {
        while (true) {
            synchronized (lock) {//或者直接把this作为锁
                if (ticket > 0) {
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    System.out.println(Thread.currentThread().getName() + ":卖票，票号：" + ticket);
                    ticket--;
                } else {
                    break;
                }
            }
        }
    }
}

如果把lock的创建放在run()内部，就不是只有一把锁了。
若是使用继承thread类的方式，则不仅要给数据加static，锁也要加static。也可以简便写MyThread.class，即将继承Thread的子类的类对象作为锁。因为每个类都有一个类对象，且唯一。

方式二：同步方法

如果需要操作共享数据的代码在一个方法内部，则可以将方法声明为同步的。

class MyThread1 implements Runnable {
    private int ticket = 100;

    @Override
    public void run() {
        show();
    }

    private synchronized void show() {
        while (true) {
                if (ticket > 0) {
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    System.out.println(Thread.currentThread().getName() + ":卖票，票号：" + ticket);
                    ticket--;
                } else {
                    break;
                }
        }
    }
}

继承类的写法如下：

class MyThread extends Thread {
    private static int ticket = 100;

    @Override
    public void run() {
        show();
    }

    private static synchronized void show() {
        while (true) {
                if (ticket > 0) {
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    System.out.println(Thread.currentThread().getName() + ":卖票，票号：" + ticket);
                    ticket--;
                } else {
                    break;
                }
        }
    }
}

同步方法实际还是和同步代码块一样用的同步监视器，只不过不需要显式的写出来，非静态的同步方法的锁是this，静态的同步方法的锁是当前类的类对象。

方式三：Lock锁

class MyThread1 implements Runnable {
    private int ticket = 100;
    private ReentrantLock lock = new ReentrantLock(true);

    @Override
    public void run() {
        while (true) {
            try {
                lock.lock();
                if (ticket > 0) {
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    System.out.println(Thread.currentThread().getName() + ":卖票，票号：" + ticket);
                    ticket--;
                } else {
                    break;
                }
            } finally {
                lock.unlock();
            }
        }
    }
}

线程间的通信

如下代码，想让两个线程交替打印，但是往往被一个线程给全部占了：

package com.ylxianzhe.thread;

public class Test1 {
    public static void main(String[] args) {
        MyThread1 myThread1 = new MyThread1();
        Thread t1 = new Thread(myThread1);
        Thread t2 = new Thread(myThread1);
        t1.setName("线程1");
        t2.setName("线程2");
        t1.start();
        t2.start();
    }
}

class MyThread1 implements Runnable {
    private int number = 1;

    @Override
    public void run() {
        while (true) {
            synchronized (this) {
                if (number < 100) {
                    System.out.println(Thread.currentThread().getName() + ":" + number);
                    number++;
                } else {
                    break;
                }
            }
        }
    }
}

让线程一让给线程二就是线程的通讯。

package com.ylxianzhe.thread;

public class Test1 {
    public static void main(String[] args) {
        MyThread1 myThread1 = new MyThread1();
        Thread t1 = new Thread(myThread1);
        Thread t2 = new Thread(myThread1);
        t1.setName("线程1");
        t2.setName("线程2");
        t1.start();
        t2.start();
    }
}

class MyThread1 implements Runnable {
    private int number = 1;

    @Override
    public void run() {
        while (true) {
            synchronized (this) {
                notify();//唤醒另一个线程，让其进入就绪状态，如果有多个，就会唤醒优先级高的，notifyAll则是唤醒所有
                if (number < 100) {
                    System.out.println(Thread.currentThread().getName() + ":" + number);
                    number++;
                } else {
                    break;
                }
                try {
                    wait();//线程一进入阻塞，执行wait()会释放锁，让线程二执行
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

notify，notifyAll，wait使用必须是在同步代码块或同步方法中。且三个方法的调用者必须是同步监视器。这三个方法定义在Object类中。

sleep与wait的异同：

同：两个调用后都会让线程进入阻塞状态。

异：sleep是在Thread中声明的，而wait是在Object中。所以调用时sleep可以在任何需要时就调用，而wait必须在同步代码块或同步方法中。wait还会释放锁，sleep不会。

其他两种创建多线程的方式

实现Callable接口

package com.ylxianzhe.thread;

import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.FutureTask;

public class Test2 {
    public static void main(String[] args) {
        NumThread numThread = new NumThread();
        FutureTask futureTask = new FutureTask(numThread);//接受Callable的返回值
        new Thread(futureTask).start();//被执行
        try {
            Object o = futureTask.get();
            System.out.println((int)o);
        } catch (InterruptedException | ExecutionException e) {
            e.printStackTrace();
        }
    }
}

class NumThread implements Callable {
    @Override
    public Object call() throws Exception {
        int sum = 0;
        for (int i = 0; i < 100; i++) {
            if (i % 2 == 0) {
                sum += i;
            }
        }
        return sum;
    }
}

1.创建一个Callable接口的实现类

2.将此线程要执行的操作写在call()方法中

3.创建Callable接口实现类的对象

4.将此实现类对象传递到FutureTask构造器

5.将FutureTask对象传递到Thread类的构造器，创建Thread的对象，并调用start方法。

该方法的优点：

1.call可以有返回值

2.call可以抛出异常

3.Callable是支持泛型的。

Future接口可以对Runnable，Callable任务的执行结果进行取消，查询，获取结果等。FutureTask是这个接口的唯一实现类，但它又同时实现了Runnable接口，所以FutureTask即用来接受Callable的返回值，也用来被线程执行。

创建线程池

ExecutorService：真正的线程池接口。常见子类ThreadPoolExecutor。

execute(Runnable command):执行任务，没有返回值，一般执行Runnable

Future submit(Callable task):执行任务，没有返回值，一般执行Callable

void shutdown():关闭连接池

Executors:工具类，线程池的工厂类，用于创建并返回不同类型的线程池

Executors.newCachedThreadPool()：创建一个可根据需要创建新线程的线程池

Executors.newFixedThreadPool(n)：创建一个可重用固定线程数的线程池

Executors.newSingleThreadPool()：一个线程的线程池

Executors.newScheduledThreadPool(n)：一个可在给定延迟后运行命令或定期执行的线程池

package com.ylxianzhe.thread;


import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.ThreadPoolExecutor;

class NumberThread implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i < 100; i++) {
            if (i % 2 == 0) {
                System.out.println(i);
            }
        }
    }
}

public class Test {
    public static void main(String[] args) {
        ExecutorService service = Executors.newFixedThreadPool(10);//ExecutorService是接口，Executors工具类返回的实际是ThreadPoolExecutor
        //ThreadPoolExecutor继承于AbstractExecutorService抽象类，它又实现了ExecutorService
        //ThreadPoolExecutor可以设置线程池的属性
        ThreadPoolExecutor service1 = (ThreadPoolExecutor) service;
        service1.execute(new NumberThread());//适合使用与Runnable,可以用匿名内部类或lambda表达式
        //service.submit();//适合使用与Callable
        service.execute(
                () -> {
                    // ... do something inside runnable task
                    System.out.println(1000);
                });
        service.shutdown();
        service1.shutdown();
    }
}
