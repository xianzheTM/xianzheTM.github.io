分类

顶级父类是Throwable，下属两个，error和exception，error不处理，主要处理exception。exception又分为编译时异常和运行时异常。

处理机制

try-catch-finally

处理异常

public class test {
    @Test//使用单元测试：方法必须是public，没有返回值，没有参数，类必须是public，构造器无参
    public void test() {
        int num=0;
        String string = "hello";
        try {
            num = Integer.valueOf(string);
            System.out.println(1);//这里不执行
        } catch (NumberFormatException e) {
            //异常类型要对应，可以写多个catch，然后进行类型匹配，匹配到一个就跳出try-catch结构
            System.out.println("转换异常");
            System.out.println(e.getMessage());//打印异常信息
            e.printStackTrace();//更详细的信息
        }catch(Exception e){//如果把有子父类关系的放颠倒了，则会出错，因为父类在上，子类无法执行到
            //...
        }finally{//可选的
            //抛出异常前一定会执行的代码，即使catch或try中有return语句
        }
        //try-catch结构结束，异常已被处理，继续执行
        System.out.println(2);//执行
    }

}

这其中还可以用throw手动抛出异常：

public class Test {
    public static void main(String[] args) {
        student student = new student();
        student.regist(1);
        System.out.println(student);
    }
}

class student {
    private int id;
    public void regist(int id) {
        if (id > 0) {
            this.id = id;
        }else {
            throw new RuntimeException("非法数据");//抛的是运行时异常，可以不处理
            throw new Exception("非法数据");//这样写就要显式的处理了，可以tc,或者throws
        }
    }
}

throws

把异常向上抛,甩锅。当某处发生异常，则会向上抛出异常对象，后面的代码不再执行。子类重写的方法抛出的异常要不大于父类的。

public class Test {
    public void method() throws IOException {
        File file = new File("hello.txt");
        FileInputStream inputStream = new FileInputStream(file);
        int data=inputStream.read();
        while (data != -1) {
            System.out.println((char) data);
            data = inputStream.read();
        }
        inputStream.close();
    }
}

选择

若父类中没有抛异常，则子类也不能抛异常。要是子类非要处理异常，则用try-catch来处理。
若几个方法调用之间有递进关系，建议整体try-catch。

自定义异常类

先继承一个异常类，Exception或者RuntimeException。
然后要有一个static final long serialVersionUID,用作标识符。
