Java集合分为Collection接口和Map接口两个体系。Collection又分为List接口和Set接口，List元素有序，可重复。Set元素无序，不可重复。Map则是键值对。

List：ArrayList,LinkedList,Vector

Set:HashSet,LinkedHashSet,TreeSet

Map:HashMap,LinkedHashMap,TreeMap,HashTable,Properties

List

Set

HashSet是主要实现类，线程不安全，可以存null值，LinkedHashSet作为HashSet子类，又链表结构，看似是有序的。TreeSet底层用红黑树实现，可以按照添加对象的指定属性排序。

Set子接口没有定义新方法。

无序性：以HashSet为例，HashSet底层是HashMap，再底层其实还是数组但是不是紧挨着排列的，具体存储位置要看Hash值，依靠这个决定在数组中的位置。

不可重复：很简单，重复的元素哈希值也是一样的。所以要重写Hash方法。

添加过程，以HashSet为例：先算哈希值，用该哈希值算出位置，若该位置上空，则直接放入，若不为空，则先用哈希值判断，若不同则用链表添加，若同再用equals判断是否相等，若相等，则不放入，若不相等，则用链表放在该位置。

所以，在类中没有重写hashcode方法，只重写了equals方法时，两个内容一样的对象可以放到HashSet中，因为算出来的hash值不一样，位置也就不一样了，直接就放进去，没有调用equals方法。

LinkedHashSet是在HashSet原有基础上为每个元素添加了一对指针域，使得遍历可以按照添加顺序来。对于频繁遍历的操作，LinkedHashSet更加高效。

TreeSet只能添加同类的元素。而且可以按照添加的元素的属性进行排序。排序分为自然排序，即自定义类要实现Comparable接口。TreeSet的自然排序是用compareTo()方法，不是equals.而定制排序需要在TreeSet的构造器中传入一个Comparator。

public void Test() {
    Person p1 = new Person(1001, "aa");
    Person p2 = new Person(1002, "bb");

    HashSet set = new HashSet();
    set.add(p1);
    set.add(p2);
    System.out.println(set);//[Person{age=1002, name='bb'}, Person{age=1001, name='aa'}]

    p1.name = "cc";
    set.remove(p1);
    System.out.println(set);//[Person{age=1002, name='bb'}, Person{age=1001, name='cc'}]

    set.add(new Person(1002, "cc"));
    System.out.println(set);//[Person{age=1002, name='bb'}, Person{age=1002, name='cc'}, Person{age=1001, name='cc'}]

    set.add(new Person(1001, "aa"));
    System.out.println(set);//[Person{age=1002, name='bb'}, Person{age=1001, name='cc'}, Person{age=1001, name='cc'}, Person{age=1001, name='aa'}]
}

remove方法会在HashSet集合中去找p1,找法就是计算哈希值，而此时p1的name被改变了，故哈希值也会改变，所以就找不到原来位置上的p1，因此remove没有删掉p1。

而后面再添加的(1001，“cc”)这个对象是使用这两个属性得到的地址值，故不会和原来的(1001,”cc”)重复，因为原来的位置是用(1001, “aa”)算的。

后来再添加的(1001, “aa”)依然能加进去。因为用哈希值算出的地址上现在已经有了(1001,”cc”)【改名字得来的那个cc】,但是这时候会用equals判断是否相同，显然是不同的，故会用链表的形式加进去。

Map

Map接口主要有（Hashtable类，子类Properties），（HashMap类，子类LinkedHashMap），（SortedMap接口，实现类TreeMap）。

HashMap是Map的主要实现类，效率高，线程不安全，而Hashtable是一个古老的实现类，线程安全，效率低，类似于Vector。HashMap是无序的，而TreeMap是有序的，另外LinkedHashMap用于遍历操作较多的时候。几个的关系和Set类似。HashMap可以添加null的key-value，而Hashtable不可以。

LinkedHashMap也是有一对指针，记录了添加顺序。

TreeMap使用key来排序，不考虑value。

HashMap底层在jdk7用的使数组加链表，jdk8又加入了红黑树。

Properties用来处理配置文件，key-value都是string。

Map添加的是entry，一个entry由一对key-value构成，无序，不可重复。key不可以重复，且无序。value可以重复，也是无序的。

HashMap里面key的对象要重写hashcode和equals方法，保证不可重复。而value的对象只要重写下equals方法。

HashMap的原理：

HaspMap map = new HashMap();

在jdk7中，在实例化后，底层创建了一个长度16的数组Entry[] table。

map.put(key1,value1);//已经添加过多次

首先计算key1的hashcode，再计算出在entry中的存放位置，如果此位置为空，则key1-value1添加成功。如果不为空，意味着该位置有一个或多个数据以链表存储（key2-value2），先用哈希值判断是否相同，若不同则用链表添加，若哈希值相同，则再用key的equals方法判断，不相等就用value1替换value2。使得该位置的元素变为key2-vlue1（这种情况的key1和key2相同，但value不一定）。

HashMap hashMap = new HashMap();
hashMap.put("aa", 12);//添加
hashMap.put("aa", 22);//修改
System.out.println(hashMap);

HashMap的扩容默认是扩容两倍。

在jdk8中，类似于ArrayList，没有直接创建长度16的数组，并且数组名字改为了Node[]，首次调用put方法时才会创建长度16的数组。当数组某一个索引位置上以链表存在的元素个数超过8且当前数组长度超过64时，此位置上的链表改为红黑树存储。

DEFAULT_INITIAL_CAPACITY:默认容量16

DEFAULT_LOAD_FACTOR:默认加载因子0.75

threshold：扩容的临界值=16*0.75=12

TREEIFY_THRESHOLD:Bucket中链表长度大于该默认值8，转化为红黑树

MIN_TREEIFY_CAPACITY:树化改造的bucket的最小值，64，即超过该值才会树化。

遍历Map，没有给Map用的遍历器。

关键是获得key构成的Set集合，value构成的Collection，entry构成的set。这样每个都可以遍历了。

map.keySet()//返回Map的Set集合
map.values()//返回Map的collection
map.entry()//返回map的entry对

TreeMap的key必须是同一类的对象，因为要按照key排序，同样是自然排序（对象实现Comparable接口）和定制排序（TreeMap构造器传入Comparator）。

Collections

Collections工具类不仅可以操作Collection，也可以操作Map。

reverse(List),对list反转

shuffle(list),使list随机排序

sort(list),根据元素的自然顺序排序

sort(list,Comparator),根据指定的comparator排序

swap(list,int,int),交换list中i和j处元素

Object max(Collection,comparator),根据指定的comparator返回最大值

Object max(Collection)返回最大值

Object minmax(Collection,comparator)根据指定的comparator返回最大值

Object min(Collection)返回最小值

上面一些涉及到最值的方法都意味着元素对象时可比较的，要实现Comparable接口

int Frequency(Collection,object),返回指定元素出现的次数

void copy(list dest,list src),把src复制到dest,注意要提前使得dest的size与src的size相等。

ArrayList dest = new ArrayList(10);//错误写法，10指定的是底层数组的长度，而不是size，即元素个数
List dest = Arrays.asList(new Object[src.size()])//创建src.size个对象，并转为List来占位

最重要的是，几大类主要都是线程不安全的，安全的又不用了，所以Collections提供了一些线程安全的方法。

synchronizedXXX方法：

ArrayList list = new ArrayList()；
List list = Collections.synchronizedList(list);//将list变为线程安全的
