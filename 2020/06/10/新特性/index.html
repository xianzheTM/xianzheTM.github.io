Lambda和方法引用

对于函数式接口，原本需要创建匿名类来重写其中的抽象方法，如下所示：

Comparator<Integer> comparator = new Comparator<>() {
    @Override
    public int compare(Integer o1, Integer o2) {
        return Integer.compare(o1, o2);
    }
};
int compare = comparator.compare(12, 13);
System.out.println(compare);

上面的代码可以缩减为：

Comparator<Integer> comparator = (o1, o2) -> Integer.compare(o1, o2);
System.out.println(comparator.compare(20, 25));

省略了方法名，方法体的大括号，return。

还可以用方法引用进一步简化。当Lambda表达式内的方法是现成的，就可以进一步省略：

Comparator<Integer> comparator = Integer::compare;

再举一个具体的例子:

package com.ylxianzhe.java8.demo;

interface StringFunc {
    String func(String string);
}

public class MethodRefDemo {
    public static String stringOp(StringFunc sf, String s) {
        return sf.func(s);
    }

    public static void main(String[] args) {
        String inStr = "lambda";
        String s = stringOp(new StringFunc() {
            @Override
            public String func(String string) {
                String result = "";
                for (int i = string.length() - 1; i >= 0; i--) {
                    result += string.charAt(i);
                }
                return result;
            }
        }, inStr);
        System.out.println(s);
    }
}

上面的代码中stringOp方法第一个参数需要一个StringFunc接口的实例，于是用匿名类写了一个，并且重写了func方法来反转字符串。现在将这一部分换成Lambda表达式：

public static void main(String[] args) {
    String inStr = "lambda";
    String s = stringOp(string -> {
        String result = "";
        for (int i = string.length() - 1; i >= 0; i--) {
            result += string.charAt(i);
        }
        return result;
    }, inStr);
    System.out.println(s);
}

这里函数式接口的实例用一个稍微长一点的Lambda来表示。现在把反转的逻辑抽出来，使Lambda短一点：

public class MethodRefDemo {
    public static String stringOp(StringFunc sf, String s) {
        return sf.func(s);
    }

    public static void main(String[] args) {
        String inStr = "lambda";
        String s = stringOp(string -> {
            return reverse(string);
        }, inStr);
        System.out.println(s);
    }

    private static String reverse(String string) {
        String result = "";
        for (int i = string.length() - 1; i >= 0; i--) {
            result += string.charAt(i);
        }
        return result;
    }
}

将反转的逻辑抽出来作为一个方法，再去掉点括号和return：

public static void main(String[] args) {
    String inStr = "lambda";
    String s = stringOp(string -> reverse(string), inStr);
    System.out.println(s);
}

写成这样后，还能继续简略。这里的reverse方法是现成的，不像上上一步中要在Lambda表达式中自己写反转逻辑。而方法引用就是为此准备的，对于Lamdba表达式中已经存在的方法可以简化为：

public static void main(String[] args) {
    String inStr = "lambda";
    String s = stringOp(MethodRefDemo::reverse, inStr);
    System.out.println(s);
}

这种是类::静态方法的引用。把上面reverse方法放到一个类中而且是非静态方法：

class op {
    public String reverse(String string) {
        String result = "";
        for (int i = string.length() - 1; i >= 0; i--) {
            result += string.charAt(i);
        }
        return result;
    }
}

public class MethodRefDemo {
    public static String stringOp(StringFunc sf, String s) {
        return sf.func(s);
    }

    public static void main(String[] args) {
        op op = new op();
        String inStr = "lambda";
        String s = stringOp(op::reverse, inStr);
        System.out.println(s);
    }
}

上面就是实例::非静态方法。下面是另一种，若Lambda参数列表中的第一个参数是实例方法的参数调用者，而第二个参数是实例方法的参数时，可以使用对象方法引用。

@Test
public void test8() {
    Comparator<String> comparator = (s1, s2) -> s1.compareTo(s2);
    System.out.println(comparator.compare("ac", "ab"));
}

这是Lambda写法，参数一s1是实例方法compareTo的调用者，而第二个参数s2是compareTo的参数：

@Test
public void test8() {
    Comparator<String> comparator = String::compareTo;
    System.out.println(comparator.compare("ac", "ab"));
}

下面是构造方法引用：

@Test
public void test9() {
    Supplier<Person> sp1 = () -> new Person();
    Supplier<Person> sp2 = Person::new;//Supplier<T>，T代表返回值
    sp2.get().info();
}

无参数的。

@Test
public void test9() {
    Function<Integer,Person> function1= id -> new Person(id);
    Function<Integer,Person> function2= Person::new;//Function<T, R>，T代表参数，R代表返回值
}

一个参数的。

@Test
public void test9() {
    BiFunction<String, Integer, Person> function1 = (name, age) -> new Person(name, age);
    BiFunction<String, Integer, Person> function2 = Person::new;//BiFunction<T, U, R>，T,U代表两个参数，R代表返回值
}

两个参数的。

与构造器引用类似的数组引用:

@Test
public void test9() {
    Function<Integer, String[]> function1 = length -> new String[length];
    Function<Integer, String[]> function2 = String[]::new;
}

Stream

package com.ylxianzhe.java8.Stream;

import org.junit.Test;

import java.util.*;
import java.util.stream.Collectors;
import java.util.stream.IntStream;
import java.util.stream.Stream;

public class StreamTest {
    //Stream自己不存储数据，只操作数据
    //Stream不改变源对象，会返回一个新Stream
    //Stream延迟执行，终止操作时才会一次执行完
    //终止操作产生的结果不能再被Stream使用
    @Test
    public void test1() {//得到stream
        List<Employee> employees = Employee.getEmployees();
        //得到一个顺序流
        Stream<Employee> stream1 = employees.stream();
        //得到一个并行流
        Stream<Employee> parallelStream = employees.parallelStream();

        int[] arr = {1, 2, 3};
        //如果是用基本数据类型来获得流，返回的就是XxxStream;如果是自定义类型的，那就是Stream<T>
        IntStream stream2 = Arrays.stream(arr);
        //包装类也是用泛型
        List<Integer> integers = Arrays.asList(1, 2, 3);
        Stream<Integer> stream3 = integers.stream();

        Stream<Integer> stream = Stream.of(1, 2, 3, 4, 5);

        //上面三种流是有限流，下面是无限流
        // Stream从0开始，经过UnaryOperator函数加2，得到一个新Stream，再次使用UnaryOperator函数加2，一直循环下去，这里加了中间操作，限制为10次
        Stream.iterate(0, t -> t + 2).limit(10).forEach(System.out::println);
        //生成10个随机数
        Stream.generate(Math::random).limit(10).forEach(System.out::println);

    }

    @Test
    public void test2() {//中间操作
        List<Employee> list = Employee.getEmployees();
        Stream<Employee> stream = list.stream();

        //筛选和切片
        stream.filter(e -> e.getSalary() > 6500).forEach(System.out::println);
        //不能在stream上接着操作，“水流一旦流过就不能回到开头去操作”，limit(3)，限制个数为前三条数据
        list.stream().limit(3).forEach(System.out::println);
        //跳过三个，若是流内元素不足，返回一个空流
        list.stream().skip(3).forEach(System.out::println);
        //distinct筛选，通过流内元素的hashcode和equals方法去重
        list.stream().distinct().forEach(System.out::println);

        //映射
        List<String> list1 = Arrays.asList("aac", "bb", "ccbb", "dgd");
        list1.stream().map(String::toUpperCase).forEach(System.out::println);

        list.stream().map(Employee::getName).filter(name -> name.length() >= 3).forEach(System.out::println);
        //fromStringToStream返回一个流，使得streamStream变成一个由stream组成的stream
        Stream<Stream<Character>> streamStream = list1.stream().map(StreamTest::fromStringToStream);
        streamStream.forEach(s -> s.forEach(System.out::println));//先遍历每个流，再遍历流内的元素
        //flatmap与map的关系：当一个数组集合A（两个元素）用add方法添加另一个数组集合B（B有三个元素），A就有三个元素，第三个是一个集合B；
        // 而用addAll方法则会把B拆开放到A中，使得A有五个元素。flatmap与addAll类似
        Stream<Character> characterStream = list1.stream().flatMap(StreamTest::fromStringToStream);//由Character组成的流
        characterStream.forEach(System.out::println);

        //排序
        List<Integer> list2 = Arrays.asList(12, 13, 5, 54, 21, 88, 56, 41, 32, 55, -4, -9, -52, 0, 4);
        list2.stream().sorted().forEach(System.out::println);//自然排序
        list.stream().sorted(Comparator.comparingInt(Employee::getAge)).forEach(System.out::println);
    }

    public static Stream<Character> fromStringToStream(String string) {
        ArrayList<Character> list = new ArrayList<>();
        for (Character c : string.toCharArray()) {
            list.add(c);
        }
        return list.stream();
    }

    @Test
    public void test3() {//终止操作
        List<Employee> employees = Employee.getEmployees();

        //匹配与查找
        //检查是否匹配所有元素
        boolean b = employees.stream().allMatch(e -> e.getAge() > 20);
        //检查是否至少匹配一个元素
        boolean b1 = employees.stream().anyMatch(e -> e.getSalary() < 5000);
        //检查是否没有匹配的元素
        boolean b2 = employees.stream().noneMatch(e -> e.getSalary() < 5000);
        //findFirst,返回第一个元素
        Optional<Employee> first = employees.stream().findFirst();
        //findAny,返回流中的任意一个元素,因为是串行流，总是第一个
        Optional<Employee> any = employees.stream().findAny();
        System.out.println(any);
        Optional<Employee> any1 = employees.parallelStream().findAny();
        System.out.println(any1);
        //统计个数
        long count = employees.stream().filter(e -> e.getSalary() > 5500).count();
        //返回工资的最大值
        Optional<Double> max = employees.stream().map(Employee::getSalary).max(Double::compareTo);
        //返回有最少工资的员工
        Optional<Employee> min1 = employees.stream().min((e1, e2) -> Double.compare(e1.getSalary(), e2.getSalary()));
        Optional<Employee> min2 = employees.stream().min(Comparator.comparingDouble(Employee::getSalary));//comparingXxx方法返回一个Comparator
        //内部迭代
        employees.stream().forEach(System.out::println);
        employees.forEach(System.out::println);
    }

    @Test
    public void test4() {//终止操作归约、收集
        List<Integer> list = Arrays.asList(1, 2, 3, 4, 5, 6, 7, 8, 9);
        //第一个参数identity为初始值
        Integer sun = list.stream().reduce(0, Integer::sum);
        List<Employee> employees = Employee.getEmployees();
        Optional<Double> reduce = employees.stream().map(Employee::getSalary).reduce(Double::sum);

        //收集：collect(Collector c)，Collector接口中方法实现决定流如何收集
        //另有Collectors实用类提供很多静态方法，可以创建常见收集器实例
        List<Employee> employeeList = employees.stream().filter(e -> e.getSalary() > 6000).collect(Collectors.toList());
        employeeList.forEach(System.out::println);

        Set<Employee> employeeSet = employees.stream().filter(e -> e.getSalary() > 6000).collect(Collectors.toSet());

    }
}

Optional类

package com.ylxianzhe.java8;

import com.ylxianzhe.java8.Stream.Employee;
import org.junit.Test;

import java.util.Optional;

public class OptionalTest {
    @Test
    public void test1() {
        Employee tom = new Employee(11, "tom", 22, 7845.23);
        Optional<Employee> optional = Optional.of(tom);//tom必须非空
        //orElse方法表示optional里对象如果不空，那就返回optional里的，如果空那就返回参数other里的
        Employee employee = optional.orElse(new Employee(44,"hh",45,8899));
        System.out.println(employee);
    }

    @Test
    public void test2() {
        Employee tom = new Employee(11, "tom", 22, 7845.23);
        tom=null;
        Optional<Employee> optional = Optional.ofNullable(tom);//现在可以为空
        //orElse方法表示optional里对象如果不空，那就返回optional里的，如果空那就返回参数other里的
        Employee employee1 = optional.orElse(new Employee(44,"hh",45,8899));
        System.out.println(employee1);
        //若other参数也是null的，则返回null
        Employee employee2 = optional.orElse(null);
        System.out.println(employee2);
    }
}
