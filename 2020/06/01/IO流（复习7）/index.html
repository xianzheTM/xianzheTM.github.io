File类

File类表示文件或者文件目录。File一般用于IO流的构造器，代表文件操作的起点和终点。

package com.ylxianzhe.File;

import java.io.File;
import java.io.IOException;

public class FileTest {
    @Test

        File file = new File("hello.txt");//相对路径，以module的路径为根

IO流

分类

根据数据单位：字节流(8bit)，处理图片等非文本文件，字符流(16bit)，处理文本文件

根据流向：输入流，输出流

根据角色：节点流（直接处理文件），处理流（对流进行处理）

四大基类：

分类 字节流 字符流
输入流 InputStream Reader
输出流 OutputStream Writer

注意表中的粗体字标出的类代表节点流，斜体字标出的类代表抽象基类

先选出一部分：

抽象基类 节点流（又叫文件流） 缓冲流（处理流的一种）
InputStream FileInputStream BufferedInputStream
OutputStream FileOutputStream BufferedOutputStream
Reader FileReader BufferedReader
Writer FileWriter BufferedWriter 
@Test
public void test4() {
    FileReader fileReader = null;
    try {
        File file = new File("hello.txt");
        fileReader = new FileReader(file);
        int read = fileReader.read();//读入的字符,如果到达文件末尾，返回-1
        while (read != -1) {
            System.out.print((char) read);
            read = fileReader.read();
        }
    } catch (IOException e) {
        e.printStackTrace();
    } finally {
        try {
            if (fileReader != null) {
                fileReader.close();
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
@Test
public void test5() {
    FileReader fileReader = null;
    try {
        //1.File类的实例化
        File file = new File("hello.txt");
        //2.流的实例化
        fileReader = new FileReader(file);
        //3.读入的操作
        char[] cbuf = new char[5];
        //返回每次读入数组的个数，文件末尾返回-1
        int len;
        while ((len = fileReader.read(cbuf)) != -1) {
            for (int i = 0; i < cbuf.length; i++) {
                System.out.print(cbuf[i]);
            }
        }
    } catch (IOException e) {
        e.printStackTrace();
    } finally {
        //4.资源关闭
        if (fileReader != null) {
            try {
                fileReader.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}

上面的代码中读取helloworld.txt，里面是helloworld十个字母，刚好读完。如果是helloworld123，那么结果就是helloworld123ld,最后一次读入时只读了三个，因此上一轮读取的数组前三个被替换，后两个还是上一轮的。所以上面的写法不合适，for循环里面终止条件应该改为i<len,每次读几个就遍历几个。
用这种写法也是Ok的：

int len = 0;
while ((len = fileReader.read(cbuf)) != -1) {
    String str = new String(cbuf, 0, len);
    System.out.print(str);
}

上面都是节点流（文件流）读入文件，下面写个节点流写入文件。

如果使用字节流读取文本文件，其实也是可以的，但是只有英文字母这样一个字母占一个字节的才能正确显示，要是中文的，以utf-8编码为例，用三个字节存的，那么就会出现乱码。

上面是用节点流处理文件，下面用处理流的缓冲流来处理节点流。缓冲流可以加快文件读取速度。

public void test8(){
    File srcFile = new File("关系.png");
    File destFile = new File("关系.png");
    copyFileWithBuffer(srcFile,destFile);
}

private void copyFileWithBuffer(File srcFile,File destFile) {
FileInputStream fis;
FileOutputStream fos;
BufferedInputStream bis = null;
BufferedOutputStream bos = null;
try {
    fis = new FileInputStream(srcFile);
    fos = new FileOutputStream(destFile);

    bis = new BufferedInputStream(fis);
    bos = new BufferedOutputStream(fos);

    byte[] buffer = new byte[10];
    int len;
    while ((len=bis.read(buffer))!=-1){
        bos.write(buffer, 0, len);
    }
} catch (IOException e) {
    e.printStackTrace();
} finally {//先关闭外层的流，再关闭内层的流
    if (bis != null) {
        try {
            bis.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
    if (bos != null) {
        try {
            bos.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
    //关闭外层流的同时内层流也会自动关闭
    //fis.close();
    //fos.close();
    }
}

处理流不仅可以套接节点流，也可以套接处理流。

java还提供了转换流用于字节流和字符流之间的转换。

InputStreamReader：将一个字节的输入流转换为字符的输入流，用于将字节，字节数组解码为字符，字符串

OutputStreamWriter：将一个字符的输出流转换为字节的输出流，用于字符数组，字符串编码为字节，字节数组

对象流：存储和读取基本数据类型和对象的数据流。

序列化：用ObjectOutputStream类保存基本数据或对象的机制

反序列化：用ObjectInputStream类读取基本数据类型或对象的机制

这两个类不能序列化static和transient修饰的成员变量

要想对象是可序列化的需要满足以下要求：

1.实现Serializable接口

2.需要有一个serialVersionUID

3.内部所有属性都要是可序列化的

class Person implements Serializable{
    private static final long serialVersionUID = -1987345788358137936L;
    private String name;
    private int age;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

@Test
public void test15() {
    RandomAccessFile raf1 = null;
    RandomAccessFile raf2 = null;
    try {
        //RandomAccessFile流
        //直接继承于Object，实现了DataInput和DataOutput，既可以做输入流也可以做输出流（同一个类，但不是同一个对象）
        //RandomAccessFile的构造器第二个参数mode可以指定对文件的访问方式
        //r:只读，rw:可读可写，rwd:多一个同步文件内容的更新，rws:多一个同步文件内容和元数据更新
        //rw模式数据不会立刻写到硬盘，rwd则是。若写数据时发生异常，rw的数据则会全部丢失，rwd写入的不会
        raf1 = new RandomAccessFile(new File("关系.png"), "r");
        //若文件不存在则新建文件
        //若已存在，写入数据时是从文件头开始覆盖，而不是整个文件覆盖，当新内容少于旧内容时会出现新旧混合的情况
        raf2 = new RandomAccessFile(new File("gx.png"), "rw");

        byte[] buffer = new byte[1024];
        int len;
        while ((len = raf1.read(buffer)) != -1) {
            //对于非空文件，write方法实际就是覆盖操作，默认是从下标0开始
            raf2.seek(new File("关系.png").length());//设定覆盖起始位置为原文件末尾，实现追加内容功能
            raf2.write(buffer, 0, len);
        }
    } catch (IOException e) {
        e.printStackTrace();
    } finally {
        if (raf1 != null) {
            try {
                raf1.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
        if (raf2 != null) {
            try {
                raf2.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}
