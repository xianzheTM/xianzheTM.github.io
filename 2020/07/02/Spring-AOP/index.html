AOP,不通过修改源代码的方式添加新功能.

AOP使用动态代理实现功能增强.如果有接口,使用JDK动态代理,如果没有接口,使用CGLIB动态代理.前者是创建接口的实现类来增强被代理类功能.后者是通过创建子类来实现功能增强.

连接点:类里面哪些方法可以被增强,这些方法成为连接点.

切入点:实际被增强的方法就是切入点

通知(增强):实际增强的逻辑部分,有五种类型,前置通知,后置通知,环绕通知,异常通知,最终通知.

切面:是一种行为动作,把通知应用到切入点的过程.例如在登录前做日志记录之中前置通知就是切面.

利用AspectJ实现AOP操作:

切入点表达式:表示对类里哪个方法进行增强

语法:execution([权限修饰符][返回类型][类的全类名]方法名称)

例子:对com.company里UserDao的add方法增强:

execution(* com.compony.UserDao.add(..)) //*表示所有修饰符,..表示方法参数,返回类型被省略

例子:对对com.company里UserDao的所有方法增强:

execution(* com.compony.UserDao.*(..))

对com.company里所有类的所有方法增强:

execution(* com.compony.*.*(..))

进行通知的配置:先在spring中开启注解扫描,使用注解创建User和UserProxy对象,在增强类上添加注解,在spring配置文件中开启生成代理对象,然后配置不同类型的通知(在作为通知的方法上添加不同类型的注解和切入点表达式).

示例:

package com.company.aopanno;

import org.springframework.stereotype.Component;

/**
 * @author xianzheTM
 * @version 1.0
 * @date 2020/7/2 16:43
 */
//被增强类
@Component
public class User {
    public void add() {
        System.out.println("add");
    }
}
package com.company.aopanno;

import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.*;
import org.springframework.stereotype.Component;

/**
 * @author xianzheTM
 * @version 1.0
 * @date 2020/7/2 16:45
 */
//增强类
@Component
@Aspect//表示生成代理对象
public class UserProxy {
    //前置通知
    @Before(value = "execution(* com.company.aopanno.User.add(..))")
    public void before() {
        System.out.println("Before");
    }

    @After(value = "execution(* com.company.aopanno.User.add(..))")
    public void after() {
        System.out.println("After");
    }

    @AfterReturning(value = "execution(* com.company.aopanno.User.add(..))")
    public void afterReturn() {
        System.out.println("AfterReturning");
    }

    @AfterThrowing(value = "execution(* com.company.aopanno.User.add(..))")
    public void afterThrowing() {
        System.out.println("AfterThrowing");
    }

    @Around(value = "execution(* com.company.aopanno.User.add(..))")
    public Object around(ProceedingJoinPoint proceedingJoinPoint) throws Throwable {
        System.out.println("环绕之前");
        //被增强的方法执行,方法可能有返回值
        Object proceed = proceedingJoinPoint.proceed();
        System.out.println("环绕之后");
        return proceed;
    }
}

After是最终通知,不管有没有异常都会执行,而AfterReturning遇到异常就不会执行.
现在调用User中的add方法结果如下:

add
AfterReturning

环绕之后

上面的代码可以做公共切入点的提取:

 @Pointcut(value = "execution(* com.company.aopanno.User.add(..))")
public void pointdemo() {

}

//前置通知
@Before(value = "pointdemo()")
public void before() {
    System.out.println("Before");
}

如果有多个增强类给一个切点,那么可以用@Order注解设置顺序.

完全注解的方式:只需要在原来的配置类上加上注解@EnableAspectJAutoProxy,表示开启Aspectj生成代理对象.
