spring对jdbc进行了封装,使用JdbcTemplate方便实现对数据库操作.

一些准备工作:

配置数据库连接池,将JdbcTemplate加入Bean管理,开启组件扫描.

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
http://www.springframework.org/schema/context http://www.springframework.org/schema/beans/spring-context.xsd">

    <context:property-placeholder location="classpath:../druid.properties"/>
    <bean id="dataSource" class="com.alibaba.druid.pool.DruidDataSource">
        <property name="driverClassName" value="${driverClassName}"/>
        <property name="url" value="${url}"/>
        <property name="username" value="${username}"/>
        <property name="password" value="${password}"/>
    </bean>

    <!--JdbcTemplate对象-->
    <bean id="jdbcTemplate" class="org.springframework.jdbc.core.JdbcTemplate">
    <!--注入dataSource,前一个是JdbcTemplate中的属性,set方法注入-->
        <property name="dataSource" ref="dataSource"/>
    </bean>

    <context:component-scan base-package="dao,service"/>
</beans>

@Override

    //language=MySQL

    int update = jdbcTemplate.update(sql, null, book.getName());
    System.out.println(update);

增删改的错做都差不多,下面看下查:

返回某个值的查询:

public int selectCount() {
    //language=MySQL
    String sql = "select count(*) from test.book";
    return jdbcTemplate.queryForObject(sql, Integer.class);
}

查询返回对象:

public Book findBookInfo(String id) {
    //language=MySQL
    String sql = "select * from test.book where id=?";
    return jdbcTemplate.queryForObject(sql, new BeanPropertyRowMapper<Book>(Book.class), id);
}

RowMapper接口类似于DBUtils里的handler,针对不同的返回值有不同的实现,这里是返回对象,所以用BeanPropertyRowMapper.

返回一个集合:

public List<Book> findAllBookInfo() {
    //language=MySQL
    String sql = "select * from book;";
    return jdbcTemplate.query(sql, new BeanPropertyRowMapper<Book>(Book.class));
}

批量添加:

public void batchAddBook(List<Object[]> batchArgs) {
    //language=MySQL
    String sql = "insert into book values (?,?)";
    jdbcTemplate.batchUpdate(sql, batchArgs);
}

批量修改和删除与此类似.

事务的操作:

在UserDaoImpl中有增加钱和减少钱的方法.而在UserService中有一个利用这两个方法的转账方法.

public void accountMoney(){
    try{
        userDAo.reduceMoney();

        userDao.addMoney();
        //没有异常提交事务
    }catch(Exception e){
        //有异常,回滚事务
    }
}

事务的添加一般在Service层.用Spring管理事务有编程式和声明式两种.前一种就是上面那样写代码,不常用.后一种有注解和xml两种实现.

Spring中的事务管理提供了一个接口:PlatformTransactionManager,它针对不同的框架实现不同的实现类.

<!--创建事务管理器-->
<bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <!--注入数据源-->
    <property name="dataSource" ref="dataSource"/>
</bean>

在spring配置文件中开启事务注解要引入名称空间tx.

<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xmlns:tx="http://www.springframework.org/schema/tx"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context.xsd
http://www.springframework.org/schema/tx http://www.springframework.org/schema/tx/spring-tx.xsd">
<!--开启事务注解-->
<tx:annotation-driven transaction-manager="transactionManager"/>

然后在service的类或者类里的方法上加上事务的注解.

@Transactional注解可以加到类上,也可以加到方法上.该注解还有几个重要的属性:

propagation:事务的传播行为,多事务方法之间进行调用,这个过程中事务是如何进行管理的.事务方法指对数据库进行变化的操作.在add方法(加了事务)中调用update方法(没加事务)时,事务如何管理,前者如何影响后者.或者没有事务的调用有事务的,或者两个都有事务之间的调用.

一共有七种行为:

required:如果有事务在运行就在这个事务内运行,否则就启动一个新事务,并在自己的事务内运行.add(有事务)调用update(没事务),update在add的事务中运行.若是add和update都没事务那就没事务.若是add没事务,update有事务就在update自己的事务中运行.

required_new:当前的方法必须启动新事务,并在自己的事务中运行.如果有事务在运行就把旧事务挂起.

supports:如果有事务就在事务中运行,否则可以不在事务中运行.

not_supported:以非事务方式执行操作，如果当前存在事务，就把当前事务挂起

nerver:当前方法不应该在事务中执行,如果有就抛异常

nested:如果有事务,当前方法就在该事务中嵌套事务内运行,否则就启动一个新事务在里面运行.

mandatory:当前方法必须在事务内运行,如果没有事务就抛异常

isolation:隔离级别

timeout:超时时间,事务提交的时限,超过时间后不提交就会回滚.默认值为-1,手动设置单位为秒

readOnly:是否只读

rollbackFor:回滚,设置出现哪些异常进行回滚

noRollbackFor:不回滚,设置出现哪些异常不进行回滚

基于xml实现事务管理,仍旧需要用xml配置事务管理器和数据源:

<tx:advice id="txadvice">
    <tx:attributes>
        <tx:method name="accountMoney" propagation="REQUIRED"/>
    </tx:attributes>
</tx:advice>
<!--配置切入点和切面-->
<aop:config>
    <!--配置切入点-->
    <aop:pointcut id="pt" expression="execution(* service.*(..))"/>
    <aop:advisor advice-ref="txadvice" pointcut-ref="pt"/>
</aop:config>

完全注解实现:

import com.alibaba.druid.pool.DruidDataSource;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;
import org.springframework.jdbc.core.JdbcTemplate;
import org.springframework.jdbc.datasource.DataSourceTransactionManager;
import org.springframework.transaction.annotation.EnableTransactionManagement;

import javax.sql.DataSource;

/**
 * @author xianzheTM
 * @version 1.0
 * @date 2020/7/3 14:39
 */
@Configuration
@ComponentScan(basePackages = "service")
@EnableTransactionManagement//开启事务
public class config {
    //创建数据库连接池
    @Bean
    public DruidDataSource getDruidDataSource() {
        DruidDataSource dataSource = new DruidDataSource();
        dataSource.setDriverClassName("com.mysql.cj.jdbc.Driver");
        dataSource.setUrl("jdbc:mysql://127.0.0.1:3306/test?rewriteBatchedStatements=true&serverTimezone=GMT");
        dataSource.setUsername("root");
        dataSource.setPassword("root");
        return dataSource;
    }

    @Bean
    public JdbcTemplate getJdbcTemplate(DataSource dataSource) {
        JdbcTemplate jdbcTemplate = new JdbcTemplate();
        jdbcTemplate.setDataSource(dataSource);
        return jdbcTemplate;
    }

    @Bean
    public DataSourceTransactionManager getDataSourceTransactionManager(DataSource dataSource) {
        DataSourceTransactionManager dataSourceTransactionManager = new DataSourceTransactionManager();
        dataSourceTransactionManager.setDataSource(dataSource);
        return dataSourceTransactionManager;
    }
}
