IOC容器

(1).控制反转,把对象的创建和对象间的调用过程交给spring管理

(2).使用IOC目的,降低耦合度

IOC原理

xml解析,工厂模式,反射

对象之间的相互调用的原始方式是直接在A类中new一个B类对象,这样耦合度太高.改进就是用工厂,在工厂中创建对象.而IOC就是用xml解析得到bean的类名,然后在工厂中根据类名创建对象,进一步降低耦合.IOC的底层就是对象工厂.spring提供两种IOC容器实现方式(两个接口):BeanFactory,ApplicationContext.前者是IOC容器的基本实现,是Spring内部使用的接口,开发人员一般不使用,后者是前者的子接口,有更多强大的子功能,开发人员使用.

BeanFactory加载配置文件后不会创建对象,使用这个对象时才会创建,ApplicationContext则是加载配置文件后就创建.

ApplicationContext的实现类:

两种配置方式

xml:

基于xml创建对象:,其中bean标签有很多属性,例如:

  • id:唯一标识
  • class:类的全路径

创建对象是默认执行的是无参数构造方法.

基于xml注入属性:

DI:依赖注入,就是注入属性,第一种方式是使用set注入.第二种通过有参数的构造器注入.

public class Book {
    private String bname;
    private String bauthor;

    public void setBname(String bname) {
        this.bname = bname;
    }

    public void setBauthor(String bauthor) {
        this.bauthor = bauthor;
    }

    @Override
    public String toString() {
        return "Book{" +
                "bname='" + bname + '\'' +
                ", bauthor='" + bauthor + '\'' +
                '}';
    }
}
<bean id="book" class="com.company.test.Book">
    <!--使用property完成属性注入,name是类里面属性的名字,value是向属性注入的值-->
    <property name="bname" value="Java入门"></property>
    <property name="bauthor" value="aa"></property>
</bean>

有参构造注入需要有一个有参构造器,然后在xml中配置即可:

<bean id="orders" class="com.company.test.Orders">
    <!--有参构造注入,可以通过name给指定属性赋值,也可以通过index索引赋值-->
    <constructor-arg name="name" value="abc"></constructor-arg>
    <constructor-arg name="address" value="chain"></constructor-arg>
</bean>

对于set注入还有种简化方式,p名称空间注入:

首先要给xml加入p名称空间:xmlns:p="http://www.springframework.org/schema/p"

然后在bean标签内设置:<bean id="book" class="com.company.test.Book" p:name="C#入门" p:author="asd"></bean>

xml注入其他类型属性:

(1).null值:

<bean id="book" class="com.company.test.Book">
    <property name="author">
        <null/>
    </property>
</bean>

(2).有特殊符号的值

在xml中转义要用到CDATA:

<bean id="book" class="com.company.test.Book">
    <property name="author">
        <value><![CDATA[
        <南京>
        ]]></value>
    </property>
</bean>

注入外部bean:

<bean id="userService" class="service.UserService">
    <!--在service中注入dao,name为属性的名称,ref为要注入的属性的bean的id-->
    <property name="userDao" ref="userDaoImpl"></property>
</bean>
<bean id="userDaoImpl" class="dao.UserDaoImpl"></bean>

与之类似的把bean写在内部:

<bean id="emp" class="bean.Emp">
    <property name="name" value="lucy"></property>
    <property name="gender" value="女"></property>

    <property name="dept">
        <bean id="dept" class="bean.Dept">
            <property name="name" value="财务部"></property>
        </bean>
    </property>
</bean>

级联赋值,在一个类中给另一个类的属性赋值:

<bean id="emp" class="bean.Emp">
    <property name="name" value="lucy"></property>
    <property name="gender" value="女"></property>

    <property name="dept" ref="dept"></property>
    <!-- name要有get方法 -->
    <property name="dept.name" value="技术部"></property>
</bean>

另一种写法就是在外部bean中给属性赋值.

xml注入集合属性:

1.数组和list类型,其中数组可用array和list标签,后者只能用list标签,还有个set集合用set标签即可.

<bean id="emp" class="bean.Emp">
    <property name="name">
        <array>
            <value>
                java
            </value>
            <value>
                C#
            </value>
            <value>
                C++
            </value>
        </array>
    </property>
</bean>

2.map类型

<bean id="emp" class="bean.Emp">
    <property name="name">
        <map>
            <entry key="aa" value="bb"></entry>
            <entry key="cc" value="dd"></entry>
        </map>
    </property>
</bean>

在集合中设置对象类型的值:用多个bean标签添加多个对象,然后在集合的标签中用ref引用这些对象即可.

创建一个集合的bean,使其它bean能够直接使用:这时要引入新的名称空间

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:util="http://www.springframework.org/schema/util"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
                            http://www.springframework.org/schema/util http://www.springframework.org/schema/beans/spring-util.xsd">
    <util:list id="booklist">
        <value>aa</value>
        <value>bb</value>
        <value>cc</value>
    </util:list>

    <bean id="Emp" class="bean.Emp">
        <property name="list" ref="booklist"></property>
    </bean>
</beans>

引入新的名称空间,然后用util:list标签即可.map,set则用util:set等即可.

FactoryBean:在spring中有两种bean,自己创建的普通bean,另一种是一种工厂bean.(不是前面的BeanFactory).普通bean:在配置文件中定义bean类型,返回的就是该类型对象.而工厂bean在配置文件中定义的类型和返回的类型不一样.

工厂bean需要实现FactoryBean接口,并实现其方法.

package bean;

import com.company.test.Book;
import org.springframework.beans.factory.FactoryBean;

/**
 * @author xianzheTM
 * @version 1.0
 * @date 2020/7/1 19:13
 */
public class MyBean implements FactoryBean<Book> {
    //定义返回bean
    @Override
    public Book getObject() throws Exception {
        Book book = new Book();
        book.setAuthor("aa");
        return book;
    }

    @Override
    public Class<?> getObjectType() {
        return null;
    }

    @Override
    public boolean isSingleton() {
        return false;
    }
}

在配置文件中还是添加MyBean.在使用时:Book book = context.getBean("MyBean", Book.class);,通过MyBean得到的是Book.

Bean的作用域:在spring中可以设置bean是单示例还是多实例,默认是单实例的.在bean标签中的scope属性设置,该属性默认值是singleton,propotype表示多实例.若属性是前者,配置文件加载时就会创建单例对象,而后者是在调用getBean方法时才会创建对象.

Bean的生命周期:①通过构造器创建Bean实例(无参构造器),②为Bean属性和对其它Bean引用赋值(set),③调用Bean的初始化方法(需要配置),④Bean使用,⑤当容器关闭时,调用Bean的销毁方法(需要配置销毁方法).

public void test1() {
    ClassPathXmlApplicationContext context = new ClassPathXmlApplicationContext("bean1.xml");
    Orders order = context.getBean("order", Orders.class);
    System.out.println(order);
    //手动销毁Bean对象
    context.close();
}
<bean id="order" class="com.company.test.Orders" init-method="init" destroy-method="destroy">
    <property name="name" value="手机"></property>
</bean>
public class Orders {
    private String name;

    public Orders() {
        System.out.println("无参构造方法");
    }

    public void setName(String name) {
        this.name = name;
        System.out.println("set设置值");
    }

    public void init(){
        System.out.println("初始化方法");
    }

    public void destroy(){
        System.out.println("销毁");
    }
}

Bean的后置处理:在调用初始化方法之前和之后会把bean实例传递给bean后置处理器的方法(如果使用后置处理器的话).

创建后置处理器需要实现接口BeanPostProcessor.里面有两个方法,对应初始化操作之前的后初始化之后的.

public class MyBeanPost implements BeanPostProcessor {
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        System.out.println("初始化之前执行");
        return bean;
    }

    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        System.out.println("初始化之后执行");
        return bean;
    }
}

然后在配置文件中加入这个Bean即可.此外,后置处理器是对所有Bean生效的,即配置文件中的所有Bean在执行初始化方法前后都会执行后置处理,处理后置处理器自身.

前面通过value或者ref来指定属性值的都是手动装配,下面使用自动装配.

自动装配是根据指定规则(属性名称或者属性类型),spring自动将匹配的属性值进行注入.

<bean id="emp" class="bean.Emp" autowire="byName"></bean>
<bean id="dept" class="bean.Dept"></bean>

autowire的常用值:byName表示根据名字注入,注入值bean的id值和类属性的名称一样.即第二行dept的bean的id和emp中属性名字一样.
byType表示根据类型去注入.使用这种方式要注意配置文件中不能有相同类型的Bean,因为不能确定装配哪个bean.在需要被注入的类Emp中有重复的属性类没关系.

使用配置文件:

原本使用数据库需要在配置文件中写明连接数据库的四大属性:

<bean id="dataSource" class="com.alibaba.druid.pool.DruidDataSource">
    <property name="driverClassName" value="com.mysql.jc.jdbc.Driver"/>
    <property name="url" value="jdbc:mysql://localhost:3306/test"/>
    <property name="username" value="root"/>
    <property name="password" value="123"/>
</bean>

现在引入外部属性文件:

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
http://www.springframework.org/schema/context http://www.springframework.org/schema/beans/spring-context.xsd">

    <context:property-placeholder location="classpath:jdbc.properties"/>
    <bean id="dataSource" class="com.alibaba.druid.pool.DruidDataSource">
        <property name="driverClassName" value="${prop.driverClass}"/>
        <property name="url" value="${prop.url}"/>
        <property name="username" value="${prop.username}"/>
        <property name="password" value="${prop.password}"/>
    </bean>
</beans>

首先要引入context名称空间,然后引入外部属性文件,然后根据属性文件中的key来获取value值.

注解

下面使用注解来实现IOC.

与创建对象相关的共四个注解:@Component,@Service,@Controller,@Repository,四个注解功能是一样的,但是习惯上是在特定的层使用特定的注解.第一个是宽泛的哪都行,最后一个是持久层用的.

使用注解方式创建对象需要引入aop这个包,然后开启组件扫描.这里需要引入context名称空间.扫描多个包可用逗号隔开.

<!--开启组件扫描-->
<context:component-scan base-package="com.company"/>

为类加上注解:

//value类似于原来的id,也可以不写,默认值为类名(首字母小写)
@Component(value = "userService")
public class UserService {
    public void add() {
        System.out.println("service add");
    }
}

开启组件扫面中的一些细节:

<!--开启组件扫描-->
<!--use-default-filters="false"表示现在不适用默认的filter,使用自定义的,以达到扫描company包下部分类的功能-->
<!--include-filter设置扫描哪些类-->
<!--type="annotation"表示根据注解来过滤,后面一句表示只扫描带Controller注解的类-->
<context:component-scan base-package="com.company" use-default-filters="false">
    <context:include-filter type="annotation" expression="org.springframework.stereotype.Controller"/>
</context:component-scan>
<!--这里仍然使用默认的过滤器,即扫描包下所有类,但是给默认过滤器加上条件,排除Controller注解的类-->
<context:component-scan base-package="com.company">
    <context:exclude-filter type="annotation" expression="org.springframework.stereotype.Controller"/>
</context:component-scan>

用注解实现属性注入:@AutoWired,表示根据属性类型注入,@Qualifer,根据属性名称注入@Resource,二者都可,@Value,注入普通类型属性,前三个注入的是对象类型的属性.

要在userService中调用userDaoImpl中的add方法,则要将后者注入给前者.首先用注解注入创建两个对象.

package com.company.dao;

import org.springframework.stereotype.Repository;

/**
 * @author xianzheTM
 * @version 1.0
 * @date 2020/7/1 22:37
 */
@Repository
public class UserDaoImpl implements UserDao {
    @Override
    public void add() {

    }
}
package com.company.service;

import com.company.dao.UserDao;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

/**
 * @author xianzheTM
 * @version 1.0
 * @date 2020/7/1 22:10
 */
@Service
public class UserService {
    //为属性注入,现在不需要set方法了
    @Autowired //根据类型注入,去找UserDao类型
    private UserDao userDao;
    public void add() {
        System.out.println("service add");
    }
}

@Qualifier注解要和@Autowired一起使用:

@Autowired
private UserDao userDao;

在装配属性为接口时,如果接口只有一个实现类,或者有多个实现类但是只给一个实现类加了注解,这种情况下能找到具体的实现类.但是如果有多个,spring就无法确定这个属性到底是哪个实现类.因此要加上@Qualifier表明属性的类的名字.

@Service
public class UserService {
    @Autowired
    @Qualifier(value = "userDaoImpl2")
    private UserDao userDao;
    public void add() {
        System.out.println("service add");
        userDao.add();
    }
}

@Resource既可以当成@Autowired,也可以当成@Qualifier使用.该注解是javax中的.

对于普通属性可以用value注解,@value(value=”属性值”).

用配置类替代配置文件:

@Configuration
@ComponentScan(basePackages = "com.company")
public class SpringConfig {
}
@Test
public void test1() {
    ApplicationContext context = new AnnotationConfigApplicationContext(SpringConfig.class);
    UserService userService = context.getBean("userService", UserService.class);
    userService.add();
}
