每个基于MyBatis的应用都是以一个SqlSessionFactory的实例为核心的。SqlSessionFactory的实例可以通过SqlSessionFactoryBuilder获得。而SqlSessionFactoryBuilder则可以从XML配置文件或一个预先配置的Configuration实例来构建出SqlSessionFactory实例。

static {//放在静态代码块,保证先得到一个SqlSessionFactory
    //获取SqlSessionFactory
    String resource="mybatis-config.xml";
    try {
        InputStream inputStream = Resources.getResourceAsStream(resource);
        SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
    } catch (IOException e) {
        e.printStackTrace();
    }

}

public static SqlSession getSqlSession(){
    return sqlSessionFactory.openSession();
}

先准备一个pojo,然后写UserDao接口,在该接口中规定一些方法.原来需要写BaseDao提供基本的curd方法,然后在UserDaoImpl中调用这些方法实现Dao接口中的方法.现在只需要一个UserMapper.xml来写sql语句即可:

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<!--命名空间绑定一个对应的Dao(Mapper接口)-->
<mapper namespace="com.company.dao.UserDao">
   <!-- id为方法名,这里返回值类型不是集合,而是集合的泛型-->
   <select id="getUserList" resultType="com.company.pojo.User">
       select * from test.user
   </select>
</mapper>

然后回到mybatis-config中注册这个mapper:

<mappers>
    <mapper resource="com/company/dao/UserMapper.xml"/>
</mappers>

然后写一个测试方法:

public void getUserList() {
    SqlSession sqlSession = MybatisUtils.getSqlSession();
    //方式一

}

使用注解不能应对一些复杂的情况.

@Select("select * from user where id = #{id};")
User getUserByID(@Param("id") int id);

为参数加上@Param注解,即为括号里的参数取别名,与查询语句里的参数对应,此时的第三个id就是形参,可以不叫id.多个参数,且类型为基本类型和string类型时就必须要加该注解.

4.按索引位置传,不推荐.

如果一个实体类中含有另一个实体类,分别对应了两张表,这个被包含的实体类需要用resultMap来特别的映射.例如,student中包含了一个teacher.

<select id="getStudent" resultMap="StudentTeacher">
    select * from student
</select>
<resultMap id="StudentTeacher" type="Student">
    <result property="id" column="id"/>
    <result property="name" column="name"/>
    <!--对于复杂的属性要单独处理
        对象:association
        集合:collection
     -->
     <!-- column表示表中字段名,property表示属性名,这样对应起来,javaType告诉这个属性的类型 -->
     <association property="teacher" column="tid" javaType="Teacher" select="getTeacher"/>
</resultMap>

<select id="getTeacher" resultType="Teacher">
    select * from teacher where id= #{id}
</select>

首先是为了查student的数据,然后为了把student中teacher的信息查出来,又在后面查了teacher,最后用resultMap将teacher的数据与student中teacher属性映射.这是按照查询嵌套处理.还有一种按照结果嵌套处理的写法:

<select id="getStudent2" resultMap="StudentTeacher2">
    select s.id sid,s.name sname,t.name tname
    from student s,teacher t
    where s.tid=t.id;
</select>

<resultMap id="StudentTeacher2" type="Student">
    <result property="id" column="sid"/>
    <result property="name" column="sname"/>
    <association property="teacher" javaType="Teacher">
        <result property="name" column="tname"/>
    </association>
</resultMap>

如果反过来,一个老师有多个学生,老师的实体类中含有一个集合.

<select id="getTeacher" resultMap="TeacherStudent">

动态SQL

if:

<select id="queryIf" resultType="com.company.pojo.Employees" parameterType="Map">
    select *
    from myemployees.employees
            where 1 = 1
    <if test="commission_pct != null">
        and commission_pct = #{commission_pct}
    </if>
    <if test="department_id != null">
        and department_id = #{department_id}
    </if>
</select>

将commission_pct和department_id作为参数放入map中传入,如果两个参数有不为null的,根据条件在第一个SQL语句后追加查询条件.这里要注意的是where 1=1后面不能有分号结束SQL语句,因为这种写法本质是SQL语句的拼接.

上面的语句的问题在于有个where 1=1这种不符合规范的东西.这里应该使用where标签.where标签可以自动将多余的and去掉.

choose:

类似于switch语句,只会根据传入的一个参数去选择查询.满足哪个条件查哪个.

<select id="queryChoose" resultType="com.company.pojo.Employees" parameterType="map">
    select *
    from myemployees.employees
    <where>
        <choose>
            <when test="commission_pct != null">
                commission_pct = #{commission_pct}
            </when>
            <when test="department_id != null">
                and department_id = #{department_id}
            </when>
            <otherwise>
                and manager_id = #{manager_id}
            </otherwise>
        </choose>
    </where>
</select>

而动态更新则用set:

<update id="updateAuthorIfNecessary">
  update Author
    <set>
      <if test="username != null">username=#{username},</if>
      <if test="password != null">password=#{password},</if>
      <if test="email != null">email=#{email},</if>
      <if test="bio != null">bio=#{bio}</if>

SQL片段:可以使用SQL标签将一段公共的SQL语句抽取出来,并给个id,需要使用时则用include标签加refid引用这个片段.在SQL片段里的语句最好基于单表,也不要有where标签.

foreach标签:

mapper中的代码List<Employees> queryForEach(Map<String, ArrayList<Integer>> map);

mapper.xml:

<select id="queryForEach" resultType="com.company.pojo.Employees" parameterType="map">
    select *
    from myemployees.employees
    <where>
        <foreach collection="ids" item="mid" open="(" close=")" separator="or">
            manager_id = #{mid}
        </foreach>
    </where>
</select>

这里使用一个mao传入参数:map(ids:list),key为ids,value为一个ArrayList数组.foreach的collection根据传入的map参数的key:ids拿到要遍历的数组,item为集合元素的代称,方便下一行把集合元素赋给查询语句.open,close,separator为拼凑SQL语句的字符.例如:

Map<String, ArrayList<Integer>> map = new HashMap<>();
ArrayList<Integer> list = new ArrayList<>();
list.add(100);
list.add(120);
map.put("ids", list);
List<Employees> employees = mapper.queryForEach(map);

集合里元素为100,120.那么拼凑的SQL为select * from myemployees.employees WHERE ( manager_id = 100 or manager_id = 120 ).

另一个例子:

<select id="queryForEach" resultType="com.company.pojo.Employees" parameterType="map">
    select *
    from myemployees.employees
            where employees.manager_id in
    <foreach collection="ids" item="mid" open="(" close=")" separator=",">
        #{mid}
    </foreach>
</select>

拼凑的结果为:

select *
FROM myemployees.employees
WHERE employees.manager_id in ( 100 , 120 );

这里使用Map将一个ArrayList传进去其实挺麻烦,可以直接把ArrayList作为参数传入.

下面是Mybatis的缓存.

mybatis也提供了对缓存的支持，分为一级缓存和二级缓存。但是在默认的情况下，只开启一级缓存（一级缓存是对同一个SqlSession而言的）。

一级缓存:

同一个SqlSession对象,在参数和SQL完全一样的情况下，只执行一次SQL语句（如果缓存没有过期）,SQL查询的结果是同一个对象.

不是同一个SqlSession对象,在参数和SQL完全一样的情况下,不会缓存.

如果在mapper.xml中设置flushCache=”true”,则会刷新缓存.同一个SqlSession中在参数和SQL完全一样的情况下还是会重新查询.

任何的UPDATE,INSERT,DELETE语句都会刷新缓存(commit)。

二级缓存:

首先要手动开启mybatis二级缓存。

在config.xml设置二级缓存开关，还要在具体的mapper.xml开启二级缓存.二级缓存是用来解决一级缓存不能跨会话共享的问题的，范围是namespace 级别的，可以被多个SqlSession 共享（只要是同一个接口里面的相同方法，都可以共享）

cache的属性简介:

eviction:代表的是缓存回收策略，目前MyBatis提供以下策略:

LRU:最近最少使用，移除最长时间不被使用的对象。(默认策略)

FIFO:先进先出，按对象进入缓存的顺序来移除对象。

SOFT:软引用，移除基于垃圾回收器状态和软引用规则的对象。

WEAK:弱引用，移除基于垃圾回收器状态和弱引用规则的对象。

flushInterval对应刷新间隔，单位毫秒，默认值不设置，即没有刷新间隔，缓存仅仅在刷新语句时刷新。

如果设定了之后，到了对应时间会过期，再次查询需要从数据库中取数据。

除此之外还有许多设置:

type用于指定缓存的实现类型，默认是PERPETUAL,对应的是mybatis本身的缓存实现类org.apache.ibatis.cache.impl.PerpetualCache。这个实现类本质是基于HashMap的.MyBatis在这个实现类基础上还提供了很多装饰器.

后续如果我们要实现自己的缓存或者使用第三方的缓存，都需要更改此处。

size对应为引用的数量，即最多的缓存对象数据，默认为1024。

readOnly为只读属性，默认为false

  • false:可读写，在创建对象时，会通过反序列化得到缓存对象的拷贝。因此在速度上会相对慢一点，但重在安全。
  • true:只读，只读的缓存会给所有调用者返回缓存对象的相同实例。因此性能很好，但如果修改了对象，有可能会导致程序出问题。

如果某些查询方法对数据的实时性要求很高，不需要二级缓存，怎么办？我们可以在单个Statement ID上显式关闭二级缓存（默认是true）：<select id="selectBlog" resultMap="BaseResultMap" useCache="false">.不影响一级缓存.

在mapper的同一个namespace中，如果有其他insert、update、delete操作后都需要执行刷新缓存操作，来避免脏读。这时我们只需要设置statement配置中的flushCache=“true“属性，就会默认刷新缓存，相反如果是false就不会了。

<select ... flushCache="false" useCache="true"/>
<insert ... flushCache="true"/>
<update ... flushCache="true"/>
<delete ... flushCache="true"/>
