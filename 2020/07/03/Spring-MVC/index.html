详细流程如下:

@RequestMapping用来建立URL与处理请求方法之间的对应关系.

package com.company.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;

/**
 * @author xianzheTM
 * @version 1.0
 * @date 2020/7/3 21:08
 */
@Controller
@RequestMapping(path = "user")
public class HelloController {
    @RequestMapping(path = "/hello")
    public String sayHello() {
        System.out.println("Hello SpringMVC");
        return "success";
    }

    @RequestMapping(path = "/test")
    public String testRequestMapping() {
        System.out.println("testRequestMapping");
        return "success";
    }
}

在类上再加一个注解就相当于加了一层目录,访问时需要这样写:http://localhost:8080/SpringMVC/user/test.

@RequestMapping的属性:path和value都是指映射的路径.method规定了该方法接受那种请求方式:@RequestMapping(path = "/hello", method = {RequestMethod.POST, RequestMethod.GET}).params规定了该方法接受的请求必须传入的参数:@RequestMapping(path = "/hello", params = {"username"}),如果请求为提供参数则不会响应..headers规定发送的请求头必须包含的部分:headers={"Accept"}

一般提交的数据spring能做到从string正确转换为需要的格式,但是你不能确定傻逼用户会往里面填什么东西,因此必然会产生类型转换错误.所以要自定义类型转换器.

第一步,实现一个接口Converter,该接口有两个泛型.public interface Converter<S, T>,S表示要被转换的类型,T表示要转成的类型.

public class StringToDateConverter implements Converter<String, Date> {

    @Override
    public Date convert(String source) {
        SimpleDateFormat format = new SimpleDateFormat("yyyy-MM-dd");
        try {
            format.parse(source);
        } catch (ParseException e) {
            e.printStackTrace();
        }
        return null;
    }
}

常用注解

requestParams:

如果前端传来的参数<a href="anno/testRequestParam?name=haha">RequestParam</a>名字为name,但是方法里写的username,这就会匹配不到.在参数上加个注解:

@RequestMapping("/testRequestParam")
public String testRequestParam(@RequestParam(name = "name") String username) {
    System.out.println("执行了");
    System.out.println(username);
    return "success";
}

这个注解有个属性是required,默认为true,表示name(value)必须是给定的值,即现在传入的参数名必须为”name”,username都不行.

ewquestBody:

用于获取整个请求体.

pathVariable:

这个注解用来获得请求中占位符位置上的参数.<a href="anno/testPathVariable/10">RequestParam</a>

@RequestMapping("/testPathVariable/{sid}")
public String testRequestParam(@PathVariable(name = "sid") String id) {
    System.out.println("执行了");
    System.out.println(id);
    return "success";
}

这种方式不使用?id=100这样子来传参数,而是用占位符.占位符中的和注解中的要一样,后面的id只是形参,可以不一样.这样做是为了实现restfule风格的url.即请求不是根据方法名来区别,要求每个都不一样,每个方法有自己的请求路径.restfule要求方法的路径是一样的,但是根据方法的请求方式来区别,即方法有不同的请求方式.如果有请求方式相同的方法,就要像上面那样用占位符将参数传过来,并且路径还不一样了,可以区分了.

但是表单只支持post get方法,其它方式不支持,所以可以用HiddentHttpMethodFilter实现更改请求的请求方式.

加在一个方法上时,表示该方法会先于控制器的方法执行:

当请求第一个方法时会先执行第二个方法.应用场景:需要将用户提交的数据传给一个Bean,但是传的数据并不能完整构建一个Bean,就可以用到.

@RequestMapping("/testModelAttribute")
public String testModelAttribute(User user) {
    System.out.println("testModelAttribute执行");
    System.out.println(user);
    return "success";
}

@ModelAttribute
public User showUser(String username) {
    System.out.println("showUser方法执行");
    User user = new User();
    user.setUname(username);
    user.setAge(20);
    user.setDate(new Date());
    return user;
}

先执行后一个方法,该方法内已经构建了一个user对象,然后方法执行完user会被传到第一个方法,用用户提交的值重写赋值.当ModelAttribute注解的方法没有返回值时,则需要用Map将值存入,然后让第一个方法根据key取出.

@RequestMapping("/testModelAttribute")
public String testModelAttribute(@ModelAttribute("abc") User user) {
    System.out.println("testModelAttribute执行");
    System.out.println(user);
    return "success";
}

@ModelAttribute
public void showUser(String username, Map<String, User> map) {
    System.out.println("showUser方法执行");
    User user = new User();
    user.setUname(username);
    user.setAge(20);
    user.setDate(new Date());
    map.put("abc", user);
}

拦截器

类似于过滤器:

多个拦截器组成的拦截器链执行顺序和过滤器也是类似的.二者的区别在于:过滤器是servlet规范中的,拦截器是spring mvc自己的.过滤器在url-pattern中配置/*后可以对所有访问拦截,而拦截器只拦截访问控制器的方法.自定义拦截器要实现HandlerInterceptor接口.

这个接口有三个默认方法,preHandle,postHandle,afterCompletion,分别对应预处理(controller方法执行前),后处理(controller方法执行后,页面跳转之前,post里可以跳转到新页面,但是success页面也会执行,只是不展现)和整个页面执行完后的处理(controller里return到success.jsp页面,该页面执行后执行这个方法).

首先配置拦截器:

<!--配置拦截器组-->
<mvc:interceptors>
    <mvc:interceptor>
        <!--要拦截的方法-->
        <mvc:mapping path="/user/*"/>
        <!--不要拦截的方法-->
        <!--<mvc:exclude-mapping path=""/>-->
        <!--配置拦截器对象-->
        <bean class="com.company.interceptor.MyInterceptor"/>
    </mvc:interceptor>
</mvc:interceptors>

拦截器实现:

public class MyInterceptor implements HandlerInterceptor {
    /**
     * 预处理,controller方法执行前
     * @param request
     * @param response
     * @param handler
     * @return true表示放行,执行下一个拦截器,如果没有就执行controller方法,false不放行
     * @throws Exception
     */
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("拦截器执行");
        return true;
    }
}

编写异常处理器

package com.company.exception;

import org.springframework.web.servlet.HandlerExceptionResolver;
import org.springframework.web.servlet.ModelAndView;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

/**
 * @author xianzheTM
 * @version 1.0
 * @date 2020/7/5 14:18
 */
public class SysExceptionResolver implements HandlerExceptionResolver {
    @Override
    public ModelAndView resolveException(HttpServletRequest httpServletRequest, HttpServletResponse httpServletResponse, Object o, Exception e) {
        //获取到异常对象
        SysException exception=null;
        if (e instanceof SysException) {
            exception= (SysException) e;
        }else {
            e=new SysException("其他错误");
        }
        //创建ModelAndView
        ModelAndView mv = new ModelAndView();
        mv.addObject("errorMsg",e.getMessage());
        mv.setViewName("error");
        return mv;
    }
}
